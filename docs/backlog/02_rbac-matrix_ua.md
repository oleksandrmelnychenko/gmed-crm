## Ключові правила доступу

- `CEO` єдина роль з реальним повним бізнес-доступом.
- `Patient Manager` працює тільки зі своїм пулом пацієнтів.
- `Interpreter` та `Concierge` не отримують повної медичної картки.
- `Billing` не бачить медичні дані, але бачить фінансові документи і підтверджені послуги.
- `Patient` бачить тільки те, що було явно опубліковано або відкрито для нього.
- `IT Admin` не працює з продуктивними медичними даними, окрім технічних або тестових сценаріїв.

## Наслідки для архітектури

- Авторизація має бути не тільки рольовою, а й контекстною: `role + assignment + data class + share status`.
- Для документів потрібні окремі прапори: `internal`, `external-shareable`, `patient-visible`, `medical-only`.
- Для кожної операції читання/зміни потрібен audit trail.
